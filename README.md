# 🍳 Thai Cuisine - ระบบจัดการสูตรอาหารและสั่งอาหารออนไลน์ (OOP + Web Frontend)

เว็บแอปพลิเคชันระบบสั่งอาหารและคลังสูตรอาหารไทยแท้ดั้งเดิมกว่า 100 เมนู พัฒนาด้วยโครงสร้างการเขียนโปรแกรมเชิงวัตถุ (**OOP ในภาษา Python**) เป็นระบบหลังบ้าน (Backend API Server) ร่วมกับหน้าเว็บเชื่อมต่อผู้ใช้ (**HTML5 + CSS3 + JS** เป็น Frontend Dashboard) ที่ออกแบบมาอย่างสวยงามและทันสมัย รองรับการทำรายการจำลองและดูสัดส่วนวัตถุดิบทั้งหมดได้แบบ Real-time

---

## 🌟 ฟีเจอร์หลัก (Key Features)

1. **การแบ่งหมวดหมู่ (Category Division)**:
   - จัดหมวดอาหารแยกเป็น 3 ประเภทหลัก: **ข้าวจานเดียว (One Dish)**, **เมนูเส้น/ก๋วยเตี๋ยว (Noodles)**, และ **แกง/ซุป (Soup/Curry)**
2. **ระบบสูตรและสัดส่วนวัตถุดิบ (Recipe & Ingredient Details)**:
   - แสดงรายการวัตถุดิบและปริมาณที่ใช้สำหรับอาหารแต่ละจานอย่างละเอียด
   - คำนวณสูตรอาหารอย่างสมบูรณ์ตามหลักสูตรโภชนาการสำหรับอาหาร 1 จาน
   - ถอดสูตรโดยละเอียดและแม่นยำตามข้อมูลภาพต้นฉบับสำหรับ 6 เมนูหลัก:
     * *คั่วพริกเกลือ*
     * *กะเพราแห้ง*
     * *ผัดกระเทียมพริกไทย*
     * *ผัดขี้เมา*
     * *ผัดผงกะหรี่*
     * *ผัดพริกแกง*
3. **ระบบสั่งอาหารและตะกร้าสินค้า (Ordering & Interactive Cart)**:
   - สั่งเมนูจานโปรดลงตะกร้าสินค้า ปรับจำนวน ปรับปรุงราคา และคำนวณยอดเงินรวมอัตโนมัติ
4. **ระบบชำระเงินจำลอง (Simulated Payment System)**:
   - **PromptPay QR Code**: จำลองการสแกนด้วยดีไซน์แอนิเมชันเลเซอร์สแกนเนอร์และตัวจับเวลานับถอยหลังในการทำรายการ 2 นาที
   - **Credit Card**: ฟอร์มกรอกข้อมูลบัตรเครดิตพร้อมตัวแสดงผลการพิมพ์ (Live Credit Card Mirroring Overlay)
   - **Cash**: ชำระเงินสดที่เคาน์เตอร์รับอาหาร
5. **เมนูยอดฮิต (Popular Menus Tracker)**:
   - ระบบเก็บข้อมูลสถิติจำนวนยอดสั่งซื้อของเมนูต่างๆ เพื่อแสดงผลเมนูยอดนิยมสูงสุด 6 ลำดับแรกแบบเรียลไทม์
6. **ประวัติการสั่งซื้อ (Order History Log)**:
   - ตรวจสอบรายการสั่งซื้อย้อนหลัง วันที่ทำรายการ ช่องทางชำระเงิน และสถานะชำระเงิน
   - แสดงผล **"ใบเสร็จรับเงิน (Receipt)"** จำลอง โดยสรุปยอดค่าใช้จ่ายและรายการสินค้าเมื่อทำรายการเสร็จสิ้น
7. **ระบบค้นหาอัจฉริยะ (Smart Search)**:
   - ค้นหาได้ทั้งจาก **ชื่อเมนูอาหาร** (เช่น *กะเพรา*) หรือ **ชื่อวัตถุดิบ** (เช่น *เส้นจันท์*) เพื่อระบุเมนูที่ต้องการได้อย่างรวดเร็ว

## 🏗️ โครงสร้างการออกแบบเชิงวัตถุ (OOP Architecture)

ระบบประกอบด้วยโครงสร้างคลาสหลัก 8 คลาสที่ประสานงานกันแบบ OOP:

- `Ingredient`: เก็บข้อมูลวัตถุดิบ (ชื่อวัตถุดิบ)
- `Recipe`: เชื่อมโยง `Ingredient` เข้ากับจำนวน (`quantity`) และหน่วยวัด (`unitName`)
- `Food`: ประกอบด้วยข้อมูลชื่ออาหาร ภาษาไทย หมวดหมู่ ราคา ยอดความนิยม และคอลเลกชันรายการสูตร (`recipeList: List[Recipe]`)
- `Category`: เก็บคลาสคอนสแตนท์สำหรับควบคุมประเภทหมวดหมู่อาหารและตรวจสอบความถูกต้อง
- `OrderItem`: ตัวแทนสินค้าแต่ละรายการในตะกร้าสั่งซื้อ
- `Order`: เก็บข้อมูลคำสั่งซื้อแต่ละรายการ เช่น รหัสคำสั่งซื้อ วันที่ ยอดเงินรวม รูปแบบชำระเงิน และสถานะการชำระเงิน
- `FoodManager` (Singleton): ทำหน้าที่บริหารจัดการข้อมูลอาหารทั้ง 100 เมนู การค้นหาตัวกรอง และการจัดอันดับเมนูยอดฮิต
- `OrderManager` (Singleton): บริหารจัดการคำสั่งซื้อ การจำลองประวัติเริ่มต้น และการทำรายการธุรกรรมชำระเงิน

## 🚀 วิธีการติดตั้งและคู่มือรันระบบ (ทุกวิธีและทุกขั้นตอน)

คุณสามารถทำการสร้างสภาพแวดล้อม ติดตั้ง และรันระบบได้ด้วยวิธีต่าง ๆ ดังต่อไปนี้:

### 📌 วิธีที่ 1: ติดตั้งผ่านสภาพแวดล้อมจำลอง (Virtual Environment - venv) [แนะนำสำหรับเครื่องทั่วไป]
วิธีนี้จะช่วยป้องกันไม่ให้เวอร์ชันของ Library ต่าง ๆ ไปปะปนกับโปรเจกต์อื่นในระบบคอมพิวเตอร์ของคุณ

**ขั้นตอนในการพิมพ์คำสั่งทีละสเตป:**

1. เปิด **Command Prompt (CMD)** หรือ **PowerShell**
2. ไปยังโฟลเดอร์ปลายทางของโปรเจกต์:
   ```bash
   cd "D:\OOP+PYTHON\Food\FOOD-RECIPE\thai-food-recipe"
   ```
3. สั่งสร้างสภาพแวดล้อมจำลอง (โฟลเดอร์ชื่อ `venv`):
   ```bash
   python -m venv venv
   ```
4. เปิดใช้งานสภาพแวดล้อมจำลอง (Activate):
   - **สำหรับ Windows (Command Prompt / CMD):**
     ```cmd
     venv\Scripts\activate
     ```
   - **สำหรับ Windows (PowerShell):**
     ```powershell
     .\venv\Scripts\Activate.ps1
     ```
     *(หากระบบแจ้งเตือนเรื่อง Execution Policy ให้รัน `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass` ก่อน)*
   - **สำหรับ macOS / Linux:**
     ```bash
     source venv/bin/activate
     ```
5. เมื่อ Activate สำเร็จ (จะมีคำว่า `(venv)` ขึ้นที่หน้าบรรทัด) ให้ทำการติดตั้งโมดูล **Flask**:
   ```bash
   pip install flask
   ```
6. สั่งรันแอปพลิเคชันหลังบ้าน:
   ```bash
   python main.py
   ```
7. เปิดเว็บบราวเซอร์ เช่น Chrome, Edge แล้วพิมพ์ Url:
   👉 **[http://127.0.0.1:5000](http://127.0.0.1:5000)**

---

### 📌 วิธีที่ 2: ติดตั้งและรันโดยตรงบนเครื่องแบบ Global (วิธีด่วน)
ใช้ในกรณีต้องการติดตั้งโมดูลตรงเข้าคอมพิวเตอร์ของคุณทันทีโดยไม่ต้องสร้าง venv

**ขั้นตอนในการพิมพ์คำสั่งทีละสเตป:**

1. เปิดบอร์ดหน้าจอรับคำสั่ง (Terminal)
2. เข้าสู่โฟลเดอร์โปรเจกต์:
   ```bash
   cd "D:\OOP+PYTHON\Food\FOOD-RECIPE\thai-food-recipe"
   ```
3. พิมพ์คำสั่งติดตั้งโมดูลลงเครื่องโดยตรง:
   ```bash
   pip install flask
   ```
4. สั่งรันแอปพลิเคชัน:
   ```bash
   python main.py
   ```
5. เปิดหน้าเว็บที่ลิงก์:
   👉 **[http://127.0.0.1:5000](http://127.0.0.1:5000)**

---

### 📌 วิธีที่ 3: การรันผ่านคำสั่ง Flask CLI (Flask Run)
เป็นการรันโดยตรงผ่านโมดูลของ Flask ที่ได้รับการดาวน์โหลดมาแล้ว

**ขั้นตอนในการพิมพ์คำสั่งทีละสเตป:**

1. เปิด Terminal และเข้าสู่โฟลเดอร์โปรเจกต์:
   ```bash
   cd "D:\OOP+PYTHON\Food\FOOD-RECIPE\thai-food-recipe"
   ```
2. พิมพ์ตั้งค่าตัวแปรระบบหลัก (Environment Variables):
   - **สำหรับ Windows CMD:**
     ```cmd
     set FLASK_APP=app.py
     set FLASK_ENV=development
     ```
   - **สำหรับ Windows PowerShell:**
     ```powershell
     $env:FLASK_APP="app.py"
     $env:FLASK_ENV="development"
     ```
   - **สำหรับ macOS / Linux:**
     ```bash
     export FLASK_APP=app.py
     export FLASK_ENV=development
     ```
3. สั่งเปิดโปรแกรมผ่านคำสั่งของ Flask:
   ```bash
   flask run --port=5000
   ```

## 🛠️ ขั้นตอนการสร้างโครงสร้างโฟลเดอร์ตั้งแต่เริ่มแรก (Recreating Directory Structure)
หากต้องการขึ้นโครงสร้างโปรเจกต์นี้ใหม่ตั้งแต่ไม่มีไฟล์ใดเลย สามารถใช้คำสั่งนี้เพื่อสร้างไดเรกทอรีโฟลเดอร์ปลายทางได้ทันที:

- **คำสั่ง PowerShell (Windows):**
  ```powershell
  # สร้างโครงสร้างโฟลเดอร์ทั้งหมด
  New-Item -ItemType Directory -Path "models", "services", "templates", "static", "static/css", "static/js", "tests"
  ```
- **คำสั่ง Command Prompt / CMD (Windows):**
  ```cmd
  mkdir models services templates static static\css static\js tests
  ```
- **คำสั่ง Terminal (macOS / Linux):**
  ```bash
  mkdir -p models services templates static/css static/js tests
  ```

## 🧪 การรันชุดทดสอบความถูกต้อง (Running Unit Tests)

โปรเจกต์นี้มาพร้อมชุดการตรวจสอบความถูกต้องด้วยยูนิตเทส (Unit Tests) เพื่อตรวจสอบลอจิกการคำนวณเงิน ตรวจสถานะสั่งซื้อ และตรวจสอบตัวกรองการสืบค้นข้อมูล
โดยรันคำสั่งตรวจสอบดังนี้:

1. เข้ามาที่โฟลเดอร์โปรเจกต์
2. พิมพ์คำสั่งรันเทส:
   ```bash
   python -m unittest discover -s tests
   ```
